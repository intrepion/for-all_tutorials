## Purpose

`employee-list-removal` exists to teach:

- working with a fixed canonical list of employee names
- removing one entry by exact name match
- returning a new filtered list instead of mutating the original input
- separating list construction, removal, and formatting
- keeping adapters thin while the core owns the employee-list rules

This project follows the shared workflow and output model in [Projects](../README.md#shared-workflow) and [Projects](../README.md#shared-output-model).

The supported compiled tutorial outputs for this project are declared in [Manifest](manifest.yaml).
