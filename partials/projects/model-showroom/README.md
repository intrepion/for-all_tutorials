## Purpose

`model-showroom` exists to teach:

- loading one canonical local 3D model asset
- choosing a deterministic initial camera orbit for a strong first impression
- enabling user orbit controls and automatic rotation
- separating deterministic scene configuration from adapter-managed rendering
- keeping adapters thin while the core owns the viewer defaults

This project follows the shared workflow and output model in [Projects](../README.md#shared-workflow) and [Projects](../README.md#shared-output-model).

The supported compiled tutorial outputs for this project are declared in [Manifest](manifest.yaml).
