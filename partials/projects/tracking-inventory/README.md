## Purpose

`tracking-inventory` exists to teach:

- collecting repeated inventory-item input
- validating required numeric value input
- persisting inventory records in a local structured data file
- parsing structured inventory records from stored data
- rendering the same inventory data as both HTML and CSV
- separating storage and prompt flow from deterministic validation and report logic
- keeping adapters thin while the core owns the inventory and export rules

This project follows the shared workflow and output model in [Projects](../README.md#shared-workflow) and [Projects](../README.md#shared-output-model).

The supported compiled tutorial outputs for this project are declared in [Manifest](manifest.yaml).
