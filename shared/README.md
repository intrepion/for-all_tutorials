# Shared

This folder is for cross-framework assets that should not live inside one implementation.

Recommended subfolders:

- `benchmarking/` for load-test scenarios and benchmark harnesses
- `security/` for security review checklists and test cases
- `fixtures/` for shared seed data and mock payloads
- `contracts/` for parity helpers and common interface expectations

This is the glue that helps keep tutorial comparisons honest.
