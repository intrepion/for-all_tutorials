# Specs

This folder defines the canonical contract for each tutorial app before any framework implementation begins.

Suggested layout:

```text
specs/level-1/guestbook/
specs/level-1/todo-list/
specs/level-2/trivia-app/
```

Each project spec should eventually describe:

- scope
- routes
- data model
- validation rules
- acceptance criteria
- benchmark contract
- security checklist
